Buatlah aplikasi berbasis web dan berbasis desktop menggunakan JFrame untuk aplikasi kasir toko dengan ketentuan sebagai berikut :
1.	Jenis Toko : Toko yang menjual kaos-kaos distro
2.	Deskripsi Toko
	Nama Toko : DistroZone
	Alamat Toko : Jln. Raya Pegangsaan Timur No.29H Kelapa Gading Jakarta
	Deskripsi Toko : Toko ini adalah sebuah toko yang menjual berbagai macam kaos distro dengan berbagai macam model, warna dan ukuran. 
	DistroZone ini dikelola langsung oleh owner nya yang menjadi admin secara langsung, akan tetapi untuk transaksi penjualan semuanya dilakukan oleh karyawan atau kasir.
	Secara offline, DistroZone ini buka mulai pukul 10.00 s.d 20.00 dan pada hari senin libur sedangkan untuk transaksi onlinenya DistroZone dapat melayani setiap hari dengan jam operasional 10.00 s.d 17.00 (tanpa hari libur)
	System pembayaran untuk transaksi offline pada DistroZone ini dapat dilakukan dengan 3 cara, yaitu :
a.	Tunai / cash
b.	QRIS
c.	Transfer Bank
	Sedangkan untuk pembelian secara online baru dapat dilakukan pengiriman barang apabila sudah terverifikasi pembayaran melalui transfer ke nomer rekening owner dengan ditambah biaya ongkos kirim juga
3.	Keterangan aplikasi
Buatlah 2 versi aplikasi, yaitu aplikasi offline (desktop) dan aplikasi online (web), dengan ketentuan sebagai berikut :
	Aplikasi Offline Based Desktop, dengan ketentuan sebagai berikut :
1.	User Aplikasi ini adalah :
a)	Admin
•	CRUD Karyawan, meliputi : id_karyawan (autogenered), nama, alamat, no._telepon, NIK, foto_karyawan, username, password
•	Bisa melihat data karyawan dan live search
•	CRUD Awal Data Kaos, meliputi : id_kaos, merek_kaos, type_kaos (lengan panjang/pendek), warna_kaos, size (XS, S, M, L, XL, 2XL, 3XL, 4XL, 5XL), harga_jual, harga_pokok, stok_kaos, foto_kaos
•	Bisa melihat detail data_kaos
•	Bisa melihat laporan keuangan, berapa transaksi yang masuk serta ada laporan rugi laba nya dengan periode tertentu (seluruh kasir)
•	Atur jam operasional
b)	Kasir 
•	Bisa melihat data kaos dan live search
•	Melakukan transaksi penjualan
•	Melihat laporan keuangan dengan periode tertentu (hanya di akun nya saja)
2.	System pembayarannya : tunai, QRIS, dan transfer bank
3.	Ada laporan penjualan dan rugi / laba
4.	Desain sebaik dan selengkap mungkin, detail program harus tercover
5.	Setiap transaksi wajib diberikan struk
	Aplikasi Online Based Web, intinya tidak jauh berbeda dengan versi web, hanya ada tambahan beberapa fitur, antara lain :
1.	Adanya user customer, dimana user ini bisa melihat stok barang-barang apa saja yang dijual di DistroZone, lengkap dengan detailnya
2.	Ada layanan customer service yang melayani sesuai dengan jam operasional toko
3.	Transaksi tidak dapat dilakukan apabila belum ada verifikasi ACC dari kasir
4.	Ada tambahan biaya ongkos kirim, tiap 1 kg bisa muat 3 kaos, kaos kurang dari 3 pcs tetap di hitung 1 kg, sedangkan tariff nya sebagai berikut :
